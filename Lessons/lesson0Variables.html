VARIABLES

If we were to take Javascript and look at it under a miscroscope, we'd see that the fundamental building block for the language, its atom, is the variable. A variable does nothing else but store information. It's a way for us to keep data that we'll be using thoughout our game. Before we get technical, let's jump right in and take a moment to reflect on what basic information we could store for, say, a game like The Legend Of Zelda.

First let's ask ourselves questions about Link. Can he swing his sword? Can he jump? Does he have the ability to shoot arrows? How many bombs does he have? Is he temporarily invincible?

These are all attributes of our gameplay which could be expressed in variables. As you can imagine, there are usually a number of variables in code, especially if we're working on a large undertaking like a complete RPG! If we're making, say, a simple game like Breakout, there aren't a lot of things we need to label, so the number of variables is a great deal inferior....maybe we'll set a variable for the velocity of the ball, which bricks have been broken and our score in addition to a couple more, but certainly not as many as you'd find in a more complex game. In other words, the more complex the game, the more likely you'll have a greater number of variables in it.

We USE variables to keep track of all the stuff in our game. Luckily, once they've been created (unless we eliminate them) , our variables linger around around forever, so we don't need to worry about creating another one from scratch every time Link dies, depletes his supply of bombs, or whatever. We call the act of creating a variable "declaring a variable" by the way.

The three types of variables we'll be learning today are: variables that deal with numbers, variables that tell us whether or not something is true or false and text-based variables. If we return to our analogy with Link, what sort of variables can we conjure up based on the gameplay?

Let's start with the most common type: the numeric type:

  • Naturally we might need to keep track of how many bombs Link is currently holding, so there's our first example.
  • We might also need to know the number of dungeons Link has currently completed.
  • We might also need a variable to tell us how many times Link needs to hit the enemy before deafeating it.

But there are some less obvious uses of variables as well:

  • We might need a variable to tell us how high Link jumps.
  • We might need another one to tell us how large the screen of our game will be.
  • And we might need yet another one to telll us the current speed at which Link is moving.

As you coninue your training as a game developer, you'll develop a keen sense of what you'll need to assign variables to in your game, so thinking like this even before writing a single line of code at this phase is really quite useful. There's no hard and fast rule about how many variables your games should have or even WHAT your variables should make reference to. There are common conventions, however, which will make your programming life a whole lot easier if you follow them.

Now let's break down the variables into different types. We said there are three major types:

- Numeric values
-True/False
-Words

The technical terms for these three types in Javascript are:

-Integers/Floating Numbers
-Booleans
-Strings

Not important to memorize these now, but just be familiar with them, because these are the technical terms and you won't find a lot of programmers who will tell you they added a "word variable", but rather a "string".

Now in a minute we're going to break down each type of variable, but first let's learn how to write one!

It's very simple, actually: we write the keyword "var", then we give it a name, then we add an equal sign, then we assign it a value. After that, we just write a semicolon to tell the computer we're done.

Five steps, very simple stuff. Here's an example:

var bombsLeft = 3;

and here's another example:

var linkHeight = 4.5;

and yet another:

var hasArrows = false;

here's an example that deals with text:

var secretPassword = "swordfish";

Note that this last one had quotation marks around it. This tells us our variable is a string (text, in other words).

Now once we write a variable, we can always, at any time change its value. Huh? In other words, if we have a variable that is set to 3, like our bombsLeft variable, we can always assign it a different value. Say we bought 10 bombs at the store - all we'd need to do is do the following:

bombsLeft = 13;

and now we have 13 bombs! Simple as pie, folks! Note that there is no need to declare the word var again. We've already created our variable and given it a name, so no need to do that again.

NAMING CONVENTIONS

Since we'll be using a lot of terms in our programming, we'll need to consider that some of them are off limits when naming our variables, since they are currently used by Javascript for other stuff or have been reserved for future versions of the language.

These are things like:

function
throw
break
for
each

...and a bunch of other words. Don't be too concerned about these right now. Just stick to names that are inherent to your game and you'll probably be all right.

Another thing you'll want to consider is this:

You can't start the names of your variables with a number or any other weird character for that matter. These MUST start with either a letter or an underscore (_). I couldn't write THIS, for example:

var 5Zelda = 50;

or this:

var ^(*@@ = 10;

or this:

var 999999 = false;

but I COULD write variables with these names:

var _underscore = 40;

or this:

var treeHugger = true;

or this:

var randomVariable = 3000;

You also CAN'T have any spaces in your variables names, so be careful!

Catch my drift? Don't freak out over these conventions, you'll rarely write any weird variables if you stick to the conventions in books, online tutorials or these charming little lessons.

Another useful little convention is to use something called "camel case", which means that when you're using more than one word in your variable names, it's a good idea to start the first letter off with a lower-case letter, then capitalizing each additional one in your varaible names. Here are some simple examples:

var camelCase;
var numberOfBottlesOfBeerOnTheWall;
var currentWeight;

This technique makes your variable names easier to read and is used quite frequently by programmers.

So now that we've got the basics under control let's look at the three options we have for making variables in Javascript:

NUMBERS

This is by far the most common type of variable you'll be using. As we mentioned before it can hold whole numbers (integers), decimals (float), positive or negative numbers. They can't hold values less than - 9007199254740992 or greater than +9007199254740992.

We can perform the 4 major arithmetical operations to them and more. We can set them to zero, get their square root, assign a variable a random number or ask for the cosine equivalent of variable's cosine value, for example.

Basically most basic mathematical operations can be done with these suckers in one way or another. We'll be using numbers EVERYWHERE, by the way, much more than the other two. The math won't be that tough to get you started making games, though.

Here are some examples of numeric variables in action:

var piValue = 3.14;
var numberOfSlicesLeft = 4;
var currentGold = 800;

All these are legit...

...and these are as well.

var numberOfDaysOff = -20;
var degreesOutside = -5.6;

BOOLEANS

Boolean values are extremely simple both to grasp and to implement. There are only two possibilities: true or false. If a variable is not true, we know it's false and if a variable is true we know that it can't be false. No maybes, ifs ands or buts! Booleans are a nice way to illustrate the state of something. If we look around our rooms, we could find a number of examples that give us information about the states of our objects. Here are some examples:

var isMidnight = false;
var lightsOn = true;
var computerPluggedIn = true;
var blindsClosed = true;

Take a minute to look around your room and find a couple of these just for fun.

STRINGS

A string is a value which refers not to numberic values, but to words, letters and symbols. We need to be careful to put quotation marks around our strings to clarify that we're indeed talking about a string and not a number. Not to confuse you, but NUMBERS can ALSO be strings, but only insofar as they are text. A number in quotation marks LOSES its numeric/mathematical value and is used only for its text value.

To illustrate this point:

var slicesOfPizzaLeft = 5;
var slicesOfPizzaLeft = "5";

are NOT the same thing. The first variable (number) can be multiplied, divided, etc. while the second variable cannot.

Strings are often used in games when we have a message we want to display for the player. RPGs and text adventures are FULL of strings! Game Over! is a string. Any text, in other words, that we see in our games, especially if that text can change over the course of the game like a score value, is most likely a string.

var gameOver = "Game Over! You Lose!";
var sinistar = "Beware, coward!";
var welcome = "Welcome to our wonderful shop! Would you like to buy some bombs?";

Keep in mind that the quotation marks can be either single(') or double("), but must be consistent and end the string with the same type of quote you started off with! You can't have this without generating an error:

var wrongUse = 'This is the wrong way to write a string";
var rightUse = 'This is the right way to write a string';

Got it?

A few last things about variables. Don't let these confuse you - we'll be seeing our fair share of this stuff soon:

A variable has a sort of territorial value. We can have global variables and local variables. Local variables can only be used in the functions they are created in, while global variables can be used anywhere. We'll talk about these later since, well, we don't even know what a function IS yet!

A variable can also equal another variable. For example, the following is completely doable in Javascipt:

var apples = 10;
var oranges = 6;
var apples = oranges;

This will give both variables the value of 6.

Variables CANNOT, however, share the same name, regardless of what type of variable they are. The following will generate an error.

var apples = 5;
var apples = 7;

And that's all there really is to know about variables for now. Next time we'll take a look at basic mathematical operations and get a chance to put our variables to work! They're a whole lot easier to grasp when you actually write them! So let's go!
