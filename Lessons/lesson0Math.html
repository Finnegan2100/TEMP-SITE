BASIC MATH IN JAVASCRIPT

Now in this second primer lesson we're going to go over the basic math operations in Javascript. As I mentioned last time, numbers play a very important part in programming and you'll be working with them quite a bit, especially the four basic arithmetic operations (addition,subtraction,multiplication and division). Take note of the following:

-The result of math operations can be stored in variables, then reused at a later time.

-You can add/subtract/multiply/divide the value of a variable with a number.

-It's even possible to plug two or more variables into an operation and add/subtract/multiply/divide those variables with each other.

Here are some visual illustrations of the above points if they weren't clear. Take a close look:

var sum = 2 + 2;

This would store 4 in our variable called sum.

var first = 2;
var second = first * 6;

This would store 12 in our variable called second.

var first = 48;
var second = 8;
var third = first / second;

This would store 6 in our variable called third.

Here are the symbols we use when performing these basic operations:

+ addition
- subtraction
* multiplication
/ division

plus the oddball which is used from time to time...

% modulus operation

Now modulus operator, instead of doing standard arithmetic operations, gives us the remainder as a final value. If the two numbers divide evenly, the result is 0. Check out this example of the modulus in action:

var answer = 25 % 4;

Here answer would be a value of 1 since 4 goes into 25 six times leaving us with a remainder of 1.

Here's another example just so this sinks in.

var answer = 15 % 6;

In this case our variable named answer would have a value of 3.

Now all these operations can be used an infinite number of times in a given statement. I could have, for example:

var longOperation = 4 + 6 + 6 + 10 + 3 + 45 + 1 + 58;

And longOperation probably equals 133, but I'm too lazy to go back and double check.

For operations in which addition, multiplication, subtraction, division and modulus are mixed, you may find it necessary to use parenthesis to specify which operations should be performed first. Here are a couple examples:

var result = 7 + 3 * 3;

Here, the browser would give you the result of 16 since multiplication and division are performed before addition and subtraction. We could, however, change this around by putting parenthesis around the first two numbers making it:

var result = (7+3) * 3;

This gives us a result of 30.

Here's another example:

var answer = 56 - 4 / 8;

This gives us 55.5, since the division is always performed first, but when we add parenthesis to the formula....

var answer = (56 - 4) / 8;

Answer gets a value of 6.5. Make sense? Remind you of math class? Let's move on.

There are also shortcuts that can be used when we increase the value of our variables.

Instead of writing THIS:

result = result + 5;

We could write THIS and get the same result:

result += 5;

Get it? In both cases we're adding 5 to the variable named result. The second is much more widely used.

And this gives us the following shortcuts for all four arithmetic operations:

result = result + 5
is the same as:
result += 5;

result = result - 5
is the same as:
result -= 5;

result = result * 5
is the same as:
result *= 5;

result = result / 5
is the same as:
result /= 5;

result = result % 5
is the same as:
result %= 5;

Another widely used shortcut is this:

result = result + 1;

This is often replaced with the easier to manage:

result++;

And the same thing exists in subtraction, by the way:

result = result - 1;

Is almost always replaced by:

result--;

So what happens when you add two strings together? Simple: they become globbed together into one string!

Take this for example (don't forget that strings need to have quotation marks around them!):

var string1 = "book";
var string2 = "mark";

var string3 = string1 + string2;

What is string 3's value? Why it's "bookmark" of course!

Remember that empty spaces between the quotation marks count as well, so if we do this:

var string1 = "book ";
var string2 = "mark";

var string3 = string1 + string2;

our result is "book mark", not "bookmark" like before. Whitespaces can be pretty useful at times!

Now what happens when you add a string to a number? Does gravity stop?

Nope! All that happens is this:

var number = 60;
var string = "th";

var answer = number + string;

What's our result?

It's a string and its value is "60th".

Just remember that you can't perform mathematical operations on strings!

THE JAVASCRIPT CONSOLE

Now let's take a minute to test these out a bit. Sure, variables and numbers are great and all, but how do we actually get to see some output? Wouldn't it be great if we could see the results of our math pop up on the screen? Everything up until now has been sort of abstract - let's get our hands dirty!

Before we can see what we output, we need two things:

1. A text editor.
2. A browser.

That's it. You can add a teaspoon of vanilla extract if you want, but you'd probably fry your motherboard.

As for the text editor, you'll find a number of 'em for free on the web. I personally use Notepad++ because, well, that's what I've always used! Here's a list of free text editors you can use:

THIS

As for the browser, I recommend Google Chrome for debugging your code.

Now, although this is a javascript primer, we need to add a few lines of HTML to give our Javascript a place to work. Don't worry, though - it's only a few lines and they'll always be the same.

Let's take this step by step, ok?

STEP 1: Download text editor.
STEP 2: Copy the following code into your new text editor:

<!doctype html>
<title> This is my experiment. </title>
<script>
console.log("I'm a weirdo!!!");
</script>

STEP 3: Save your file with whatever name you please. Just do it on the desktop for now. Make sure you specify the .html extension.
STEP 4: Launch your file in your browser. In Notepad++ there's an option to do so under "run".

Aaaaand nothing happens!

That's because the console.log() stuff only appears in the debugging console of our browsers!

How do we get there? See step 5...

STEP 5: Click on the three horizontal lines at the top-right of Chrome (if you're using Chrome).
STEP 6: Click on tools.
STEP 7: Click on Javascript Console at the very bottom.

And THERE you'll find you console.log() output.

Not all that complicated, right?

Now that the two of you are acquainted, I can see that this is going to be the start of a beautiful relationship! The Javascript Console is not only great for making funny words pop up, but is also great for DEBUGGING our Javascript when we have errors in our code.

We'll look at this whole debugging thing as we proceed, but for now let's use the Javascript Console to output the results of our math operations. This way we can at least see what we're doing.

Oh, and VERY important: all the Javascript we'll be writing must go BETWEEN the <script> and </script> tags in our html! Note that in the code we just wrote, this is the case. If you put your Javascript anywhere else (for now, at least), it WON'T work! So be careful!

So that's basically it for today - let's let this stuff sink in. I STRONGLY ADVISE you to experiment with outputting some numbers, strings and booleans to your javascript console by using console.log(). Remember that you don't need to output just one variable, but can output a whole bunch of 'em by separating them by commas. You can also directly output a string by writing something like this:

console.log("Hey, it's me!");

Experiment for now and when you're ready, go ahead and try the next lesson!
